# Product intent

Aictx Memory is local-first project memory for AI coding agents. Its public promise should stay broad rather than limiting the product to one persona: agents do better work when they can load durable project context, avoid repeated briefing and context waste, and leave memory reviewable or observable in the repo.

Primary public surfaces should help visitors recognize recurring situations where Memory starts paying off: repeated agent sessions on the same repo, growing agent-built projects, scattered decisions across chats/docs/instruction files, multiple agent tools sharing one project, and useful discoveries that should survive as project state.

Maintain this synthesis when the project's purpose, user promise, or product direction changes.
